# 개발 과정 기록 (문제 → 해결)

Go 기반 리눅스 소프트웨어 라우터 `home-router`를 만들면서 마주친 문제와
그 해결 과정을 단계별로 정리한 문서다. 내용은 git 커밋 히스토리와 실제 diff를
근거로 재구성했다. (`init` 2026-03-18 ~ 2026-03-30)

## 1단계 — 네트워크 기반 구축

### 문제 ① IP 주소 중복 설정 시 크래시 (`fc1d130`)
- LAN 인터페이스에 IP를 붙일 때(`netlink.AddrAdd`) 이미 같은 IP가 존재하면
  에러로 프로세스가 죽음.
- **해결:** `unix.EEXIST`를 감지해 `AddrReplace`로 대체 처리.
- 부수 개선: 디버깅이 안 되던 `log.Fatalln` → `log.Fatalf`로 변경해
  **에러 원인을 로그에 포함**(관측성 개선).

## 2단계 — DHCP/DNS 핵심 기능과 안정화

### 문제 ② 재시작하면 클라이언트 IP가 바뀜 (`59114d8`)
- **해결:** DHCP 임대를 JSON 파일로 **영속화**하여 재시작 후에도 IP 유지.
  MAC별 고정 임대(Static Lease) 도입.

### 문제 ③ 만료된 임대가 풀을 잠식 (`6c00afd`)
- **해결:** 만료 IP 정리 로직 추가.

### 문제 ④ DHCP IP 충돌 — 가장 중요한 안정성 이슈 (`879acc8`)
- 새 IP를 할당했는데 네트워크에서 이미 사용 중인 경우 충돌 발생.
- **해결:** 할당 전 **ARP 프로브**로 실사용 여부 확인.
  충돌 IP는 `DeclinedIPs` 파일로 영속화해 재시작 후에도 기억.

### 문제 ⑤ DNS 캐시 통계의 lock 경합 (`879acc8`)
- hits/misses 카운터를 올릴 때마다 write lock을 잡아 성능 저하.
- **해결:** `atomic.Uint64`로 전환해 **WLock 제거**.

### 같은 커밋(`879acc8`)의 추가 수정
- DHCP 임대 갱신 시 **만료시간 업데이트 누락 버그** 수정.
- DHCP RELEASE 메시지 처리 누락 보완.
- WAN DHCP 유니캐스트 갱신 실패 → **브로드캐스트로 단순화**(실패 케이스 회피).

## 3단계 — Web UI와 운영 편의성

### 문제 ⑥ 설정을 코드/파일로만 변경해야 함 (`fd2b67c`, `35e3519`)
- **해결:** REST API + React 대시보드 추가. API 변경사항(포트포워딩·정적임대·
  화이트리스트)을 **config.yml에 atomic write로 영속화**.

### 문제 ⑦ 공인 IP가 바뀌면 외부 접속 불가 (`35e3519`)
- **해결:** DDNS(Cloudflare/DuckDNS/Custom) 자동 갱신.

### 문제 ⑧ "누가 어디에 접속 중인지" 가시성 부족 (`35e3519`, `b10ca2e`)
- **해결:** conntrack 기반 트래픽 추적, Reverse DNS(IP→도메인),
  GeoIP 국가·조직(ISP) 정보. 도메인이 없는 IP는
  **domain > org > ip 우선순위**로 fallback 표시.

## 4단계 — 보안/인증 정합성

### 문제 ⑨ 프론트엔드가 인증 상태를 "쿠키 존재 여부로 추측" (`f35d26e`)
- 쿠키만 보고 로그인됐다고 가정 → 만료/위조 세션을 구분 못 함.
- **해결:** `/api/auth/check` 엔드포인트로 **서버에서 세션 유효성 검증**.
  프론트는 확인 방식으로 전환 + 로딩 상태 처리 추가.

## 한눈에 보는 문제 해결 패턴

| 패턴 | 적용 사례 |
|------|-----------|
| 영속화로 재시작 견디기 | DHCP 임대, DeclinedIPs, config.yml |
| "추측" 대신 "실측/검증" | ARP 프로브(IP 실사용), 서버 인증 검증(쿠키 추측 탈피) |
| 실패 경로 단순화 | WAN 유니캐스트 갱신 → 브로드캐스트 |
| 동시성 최적화 | DNS 캐시 카운터 atomic 전환 |
| 관측성 우선 | `Fatalf`로 에러 원인 노출, 트래픽/GeoIP 모니터링 |

전반적으로 **"기능 추가 → 실환경에서 깨지는 엣지케이스 발견 →
영속화·검증·동시성으로 단단하게"** 흐름으로 발전했다.
특히 `879acc8`(버그 수정 모음)이 안정화의 분수령이었다.
